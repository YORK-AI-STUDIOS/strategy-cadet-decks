Part 11 · Methodology and Citations · Three-Phase Methodology Pipeline

Six specialist agents, seven optional extension modules, three discovery phases. Phase 1 gathers and grounds. Phase 2 synthesises and ideates. Phase 3 cross-references and corrects. Deterministic Python orchestration throughout, not LLM-interpreted prose.

Phase 3 structural moat (one of two)
Phase 3 cross-reference caught a critical discrepancy before this deck was built: the discovery source named PwC as Fleetwood's auditor in the corporate forensics section. Phase 3 cross-reference against the Fleetwood FY25 Annual Report [2] confirmed the actual auditor is Ernst and Young Australia, signing partner Fiona Drummond, Perth. The correction was applied to the source report before the deck was built. This is one of Strategy Cadet's two structural advantages over generic brand-intelligence tools: Phase 3 catches factual errors before they reach the strategist. Verified
Orchestration: deterministic Python, not prose
The orchestration layer is deterministic Python (cadet_run.py v1.0), not LLM-interpreted prose. Each phase ran as a fresh subprocess with explicit inputs, outputs, checkpoints and resumability. The combination of local AI models and frontier API models provides synthesis capability; the Python layer provides deterministic flow control, preventing silent phase-skipping. The second structural moat: goal sanity-check in, which surface-tests the recommended strategic goal against three independent frames before committing to the business plan.
